<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend → apis.tollguru.com | Server-side outbound HTTP carrying the secret `x-api-key`; response is parsed and only `{cost,isEstimate}` re-enters the app |
| client → POST /trips/calculate | JWT-guarded (existing); request carries no secret; only toll cost/flag returned |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | `TOLLGURU_API_KEY` leakage to client | mitigate | Key read via `ConfigService` server-side only; never placed in any response object; e2e (Task 3) asserts response has no `x-api-key`/key field (TOLL-06) |
| T-01-02 | Tampering / SSRF | Outbound TollGuru URL | mitigate | Endpoint is a hard-coded constant; polyline is server-generated from Mapbox geometry, never user-supplied as a URL |
| T-01-03 | Denial of Service | Free-tier quota exhaustion (~500 req/mo) | mitigate | 30-day in-memory cache keyed by polyline hash; silent fallback to heuristic on 429/timeout (D-03). Per-process cache caveat documented (RESEARCH Pitfall 4) |
| T-01-04 | Cryptography misuse | SHA-1 cache key | accept | SHA-1 used only as a non-security content hash for the cache key, not for secrecy or integrity |
| T-01-05 | Denial of Service (external) | TollGuru slowness blocks trip calc | mitigate | `AbortSignal.timeout(8000)` + silent fallback so an external failure never breaks calculation |

No npm/pip/cargo installs in this plan (inline encoder, no `@mapbox/polyline`) — no `T-01-SC` package-legitimacy gate required.
</threat_model>
